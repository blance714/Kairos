# Kairos 时间管理应用 - 需求文档

## 项目概述

Kairos 是一款个人定制的 iOS 时间管理应用，目标是通过弹性规则控制成瘾性应用的使用，而不是简单粗暴的硬性限制。

核心设计原则：**规则要让人愿意遵守，而不是想要绕过或卸载。**

## 管控应用

| 应用 | 分类 |
|------|------|
| 小红书 | 通用组 |
| 微博 | 通用组 |
| Bilibili | 通用组 |
| 小说软件（具体 App 待定）| 独立 |

## 模式定义

### 1. 早晨模式 🌅

**触发条件**：
- 晚上模式期间开启睡眠专注，再关闭后不足 1 小时

**规则**：
- 管控应用**完全锁定**，不可使用

**目的**：防止刚醒来就刷手机

**退出条件**：睡眠专注模式解除满 1 小时 → 进入普通模式（或恢复晚上模式）

**误触恢复**：早晨模式期间重新开启睡眠专注 → 恢复睡眠状态，再次关闭时重新计时

---

### 2. 普通模式 ☀️

**触发条件**：
- 不满足早晨模式条件
- 且 不满足晚上模式条件

**规则**：
- 连续使用 15 分钟后 → 进入冷却
- 冷却 30 分钟后 → 可再次使用
- 循环

**说明**：这是白天/在外的默认模式，允许间歇性使用但防止沉迷

---

### 3. 晚上模式 🌙

**触发条件**：
- 当前位置在家（GPS 定位，出租屋范围）
- 且 当前时间 ≥ 22:00
- 且 连续 30 分钟未使用任何管控应用

**规则（额度制）**：

| 分类 | 额度 |
|------|------|
| 小说软件 | 独立 45 分钟 |
| 通用组（小红书+微博+B站）| 共享 20 分钟 |

- 用完哪个锁哪个，互不影响
- 额度当日有效，次日重置

**激活前**：在家 + 22:00 后但未满足 30 分钟未使用 → 仍为普通模式（15/30 循环）

**设计意图**：
- 洗澡自然会触发"30分钟不用"，从而激活额度
- 不强制洗澡，但用冷却期的无聊推动用户去洗
- 小说需要沉浸感所以独立额度，其他碎片化应用共享额度

**与早晨模式的交互**：
- 晚上模式期间开启睡眠专注再关闭 → 进入早晨模式
- 早晨模式结束后若仍满足晚上模式条件 → 恢复晚上模式，已消耗的额度不重置

---

## 模式优先级

```
早晨模式 > 晚上模式 > 普通模式
```

即：如果同时满足多个条件，按此优先级判断。

---

## 状态流转图

```
                    ┌─────────────────────────────────┐
                    │                                 │
[普通模式] ──在家+22:00后+30分钟未使用──▶ [晚上模式（额度制）]
     ▲                                       │
     │ 满1小时                          开启睡眠专注
     │                                       │
     │              ┌──误触重开──┐            ▼
     └──[早晨模式]◀─┘           └──▶ [睡眠专注中]
              ▲                          │
              └────── 关闭睡眠专注 ───────┘
```

说明：
- 普通模式是默认状态
- 晚上模式只有一种（额度制），不再有"未激活"阶段
- 早晨模式只从晚上模式经睡眠专注触发，白天误触不会进入
- 早晨模式结束后，根据条件回到普通或晚上模式

## 暂不实现的功能

以下功能在讨论中被否决，记录原因以备后续参考：

| 功能 | 否决原因 |
|------|----------|
| 一次性延长 30 分钟（消耗晚上额度）| 短视使用后晚上没得用 → 烦躁卸载 App |
| 查东西通行证 | 很容易变成绕过冷却的万能借口 |
| 晚间 Checklist | 便利贴更实用，且手机充电时看不了手机 |

---

## 备注

- 这是第一版硬编码实现，规则写死针对当前生活模式
- 使用过程中记录"如果能 XX 就好了"的时刻
- 积累真实需求后再考虑配置化和通用化
